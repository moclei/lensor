Key Improvements (v1.0.2+)

  • No recording indicator ‚Äî uses captureVisibleTab instead of MediaStream
  • Proper memory cleanup ‚Äî old ImageBitmaps explicitly closed before replacement
  • Simpler architecture ‚Äî no MediaStream/ImageCapture objects to manage
  • Lower idle resource usage ‚Äî no persistent video capture pipeline
  • Rate limited to 2 captures/second (Chrome enforced)
  • Peak memory ~24MB during capture, settles to ~8MB
